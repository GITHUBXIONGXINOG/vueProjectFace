$nextTick

  • Vue是异步渲染的框架
  • data改变后,DOM不会立即渲染
  • $nextTick会在DOM渲染后被触发,以获取最新的DOM节点
  • 例子
    • 定义template, 里面有一个列表

 

  • 在data中定义了list的数据

 

  • 定义点击触发的函数

 

  • 默认情况下
  • 点击添加一项后
  • 再次点击后
  • 可以看到本来应该是获取到ul子节点的长度,点击添加后长度应该是6,但显示3,再次点击应该是9,但显示6

    • 在函数中立即改变list
    • 此时立即获取dom元素是旧数据
    • 然后异步渲染,渲染完后,下一次点击时,又获取到上一次的长度6
  • 期望的是点击后立即拿到新的dom数据
  • 使用$nextTick
  • 如果是同步的话修改data一次就要渲染一次,可能会造成卡顿
  • 使用异步渲染,修改数据后做整合,只需要一次渲染提升体验

mixin

  • 多个组件有相同的逻辑,抽离出来
  • mixin不是完美的解决方案,有一些问题
  • Vue3提出Composition API旨在解决这些问题

 

 

--------------------------------------------------------------------------------------------------------------------------

使用

  1. 导入/映射/使用mixDemo组件
  2. 在mixDemo组件中导入mixin
    2. mixin.js导出一个对象
  3. 在export default 中设置mixins

 

  2. 输出

 

 

 

--------------------------------------------------------------------------------------------------------------------------

mixin

  • 将信息通过export dafault暴露出来
  • 在需要的地方导入,并使用mixins:[name, name2]
  • 这样信息就会被混合到需要的地方,混合的组件可以使用导入的属性和方法
  • 多个组件中使用相同的属性或方法,就可以使用mixin,这样只需要抽离出来,然后需要的地方直接通过mixins引用
  • 也只需要修改一个地方, 其它引用的地方全部生效

--------------------------------------------------------------------------------------------------------------------------

mixin的缺点

  • 变量来源不明确,不利于阅读
    • mixin引入的变量,并不能直接看到它的定义,变量多时就会混乱
  • 多个mixin可能会造成命名冲突
  • mixin和组件可能出现多对多的关系,复杂度较高
    • 多个组件使用一个mixin,一个组件使用多个mixin

如何理解MVVM

2021年3月30日

21:30

组件化基础

  • 很久以前就有组件化
  • 数据驱动视图(MVVM, setState)

 

----------------------------------------------------------------------------------------------------------------

很久以前的组件化

  • asp,jsp,php已经有组件化了
  • nodejs中也有类似的组件化

----------------------------------------------------------------------------------------------------------------

 

对比Vue React的组件化

  • Vue和React的组件化和之前有本质的区别
  • 区别就是数据驱动视图

 

----------------------------------------------------------------------------------------------------------------

数据驱动视图

  • 传统组件,只是静态渲染,更新还要依赖于操作DOM
  • 数据驱动视图 - Vue MVVM
  • 数据驱动视图 - React setState
  • 数据驱动视图
    • 不需要操作DOM
    • 想要改什么地方就直接修改Vue和React的数据
    • Vue和React框架本身根据数据重新渲染视图
    • 使用Vue和React开发时更加关注于数据,业务逻辑,而不是操作DOM,浏览器兼容性这些问题

 

 

----------------------------------------------------------------------------------------------------------------

Vue MVVM

  • View 视图
  • Model 模型
    • 组件的data, vuex的数据

        // Array.prototype.push.call(this, ...arguments)

function defineReactive(target, key, value) {

    // 深度监听

    observer(value)

    // 核心 API

    Object.defineProperty(target, key, {

        get() {

            return value

        },

        set(newValue) {

            if (newValue !== value) {

                // 深度监听

                observer(newValue)

                // 设置新值

                // 触发更新视图

                updateView()

            }

        }

    })

}

    if (typeof target !== 'object' || target === null) {

    }

    }

    for (let key in target) {

        defineReactive(target, key, target[key])

    }

new Object的原理

2021年4月3日

21:56

new

new func()主要过程如下：

  1. 创建一个空对象obj；
  2. 将该空对象的原型设置为构造函数的原型，即obj.__proto__ = func.prototype；
  3. 以该对象为上下文执行构造函数，即func.call(obj)；
  4. 返回该对象，即return obj。

对于第3、4步还有个小细节，如果第3步func有返回值且返回值为对象，则第4步会返回func的返回值，反之则默认返回obj。

模仿new原理的代码如下：

 

function new2(func) { // func为某个构造函数

  var createObject = Object.create(func.prototype); // 以构造函数的原型对象为原型，创建一个空对象，即创建一个{ __proto__: func.prototype }

  var returnObject = func.call(createObject); // 使用刚创建的空对象作为上下文(this)执行构造函数

  if (typeof returnObject === 'object') { // 若构造函数有返回对象，则返回该对象

    return returnObject;

  } else { // 若构造函数未返回对象，则返回Object.create创建的对象

    return createObject;

  }

};

 

Object.create()

在模仿new原理的代码中用到了Object.create()，它的作用是以入参为原型创建一个空对象，即

Object.create = function (obj) {
  return { '__proto__': obj};
};

或

Object.create = function (obj) {
  function F() {}
  F.prototype = obj;
  return new F();
};

本文所说的前端中的tree-shaking可以理解为通过工具"摇"我们的JS文件，将其中用不到的代码"摇"掉，是一个性能优化的范畴。具体来说，在 webpack 项目中，有一个入口文件，相当于一棵树的主干，入口文件有很多依赖的模块，相当于树枝。实际情况中，虽然依赖了某个模块，但其实只使用其中的某些功能。通过 tree-shaking，将没有使用的模块摇掉，这样来达到删除无用代码的目的。

  • Commonjs动态引入,不允许到代码时不知道到底有没有被引入,不能Tree-Shaking
  • ES6 Module,在编译时就知道哪些被引入,哪些没有,就可以实现Tree-Shaking

        这个库将会模拟一个完全的 ES2015+ 的环境。
